企業間の決済は、金額や取引先がそのまま外部に見えてしまうことで重大なリスクを抱えています。特にクロスボーダーの送金では、相手国やサプライチェーンの状況を推測され、競争上の不利や交渉力の低下を招く恐れがあります。取引の秘匿は贅沢ではなく必然です。しかし従来のパーミッションレスなミキサーは、不正利用の温床となり、法人が使える選択肢ではありません。企業に求められるのは、規制に従いながらも経営上の機密を守れる、安全で監査可能なプライバシー決済です。

私たちが提案するのは、Japan Smart Chain の Kaigan テストネット上に構築するシールドプールです。基盤は Tornado Cats の最小骨格で、入金時にクライアントで秘密の note を生成し、チェーン上にはコミットメントとナルファイアだけを記録します。note を知っている者だけがゼロ知識証明で引き出せる、あの仕組みを土台にします。さらに、出金のためには資格が必要です。個人は Mizuhiki Verified SBT、法人は ZK Email による DKIM 証明から発行される UNTI（ERC-6268）。オンチェーンのプライバシーポリシーは固定し、送金者と受取者の双方が Mizuhiki か UNTI を必ず保有していなければ取引は成立しません。

この仕組みを業務に馴染ませる工夫が、引き出し専用URLです。入金時にアプリが note をもとに URL を発行し、請求書やメールに添付して相手に渡せます。URL には note やプール情報がフラグメントとして埋め込まれ、クライアント側でのみ解釈されるためサーバには残りません。ワンタイム表示、QR コード、宛先バインドといった実務向けのオプションも用意できます。受取側は URL を開いてローカルで証明を生成し、同時に Mizuhiki または UNTI の保有をチェックされます。note を知っていることと資格を持っていること、この二つを満たして初めて資金が解錠されます。万一 URL が漏れても、資格がなければ引き出せないため、実務で必要なセーフティが確保されます。

内部設計はシンプルです。固定デノミネーションで金額のリンク性を減らし、deposit で sender を、withdraw で receiver を必ず検査します。ナルファイアによる二重引き出し防止も維持します。オンチェーンのポリシーは常に「KYC または KYB を双方が持っていること」に固定しており、混乱を避けています。将来的には Privacy Pools の良性由来集合を組み込み、由来がクリーンであることを匿名で証明できる拡張を目指します。

Kaigan を選ぶ理由は明快です。Mizuhiki がネイティブに存在し、Ethereum 等価のため既存ツールチェーンをそのまま使え、オンショア検証のガバナンスが企業利用に説得力を与えるからです。デモでは、入金から引き出しURLの発行、受取側での開封と資格検査、出金、そしてエクスプローラ上で送金元と出金先の相関が見えないことを実演します。フェイルケースとして、資格を持たないアドレスでの出金が拒否される様子も示します。次の段階では、Association Set の導入やリレイヤー自動化、ステーブルコイン対応を進め、企業の財務や購買にとっての「当たり前の支払い体験」に近づけます。

このシールドプールは、商用・クロスボーダー決済の現実的な解答です。見られないが、守られている。資格を備えた当事者同士だけが安全にやりとりできる、そのためのプライバシー決済レイヤを Kaigan 上で始めます。
